sky-EXPID¶

Summary

Placeholder data model for the sky monitor raw data

Naming Convention

sky-EXPID.fits.fz, where EXPID is the zero-padded 8-digit exposure ID.

Regex

sky-[0-9]{8}\.fits\.fz

File Type

FITS, 70 MB

Contents¶

Number

EXTNAME

Type

Contents

HDU0

SKY

Empty HDU

Header keywords only

HDU1

SKYCAM1

Compressed IMAGE

Sky camera frames

HDU2

SKYCAM1T

BINTABLE

Metadata about each frame

HDU3

SKYCAM0

Compressed IMAGE

Sky camera frames

HDU4

SKYCAM0T

BINTABLE

Metadata about each frame

The SKYCAMERA data will be 3D[nframes, ny, nx] such that data[i] is the 2D sky camera frame number i. Row i of the SKYCAM[01]T table will contain the metadata about that frame, e.g. the DATE-OBS and EXPTIME.

Needs confirmation: SKYCAM1 and SKYCAM0 may appear in any order? Will there ever be any HDUs besides these?
